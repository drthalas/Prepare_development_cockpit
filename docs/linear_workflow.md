# Linear Workflow

## Project

Linear project: `Prepare Development Cockpit`.

The Linear roadmap contains tasks `PDC-001` through `PDC-022`.

## Phase 0

Phase 0 is `Product & SaaS Foundation` and includes:

- `ALE-160 / PDC-001 Product Definition & Repository Foundation`.
- `ALE-161 / PDC-002 UI System & Visual SaaS Shell`.
- `ALE-162 / PDC-003 Railway Deployment Baseline`.

## Source Of Truth

Linear is the roadmap and task source of truth. Each Linear task should contain a Codex Prompt that defines the implementation scope for that task.

## Execution Model

- Work tasks in phase order.
- Keep each task scoped.
- Commit each task separately.
- Run checks before reporting completion.
- Review at the end of each phase before starting the next phase.

## Review Checkpoints

Between phases, review:

- Product direction.
- Architecture changes.
- Deployment status.
- Known risks.
- Updated task scope for the next phase.
